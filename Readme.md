### 2. MVC (Model–View–Controller) Architecture

MVC (Model–View–Controller) is a software design pattern used to organize and structure applications. It divides an application into three main components, making development, testing, and maintenance more efficient.

* **Model:** Manages application data and business logic.
* **View:** Displays data and provides the user interface.
* **Controller:** Processes user input and coordinates communication between the Model and View.
* Promotes a clear separation of responsibilities within an application.
* Improves code organization, maintainability, and scalability.
